# ABTest Daily Analysis Prompt

## 用途

用于让 Codex 分析 ABTest 日报、历史 Excel/JSON、实验指标和图片解锁相关数据，产出 2 分钟内能读完的管理层判断和行动清单。

## 模板

```text
你这次的角色是：实验分析师 + 自动化工程师。

目标：
- 基于下面的 ABTest/日报数据，给出今天是否放量、继续观察、暂停/回滚或补数后判断。

输入材料：
- 项目/实验名称：
- 日期范围：
- 今日数据：
- 历史对比：
- 口径说明：
- 已知异常：

请先做：
0. 如果涉及脚本、调度、登录态、补跑、发送或自动化改动，先按 `daily-work-task-card.md` 判断 Light / Full、证据、门禁和外部动作边界。
1. 检查数据是否完整，尤其是收入、转化、样本量、实验组/对照组口径。
2. 判断是否存在不能直接下结论的异常。
3. 如果数据足够，给出明确决策；如果不足，给出补数项。

输出格式：
1. 一句话结论
2. 关键指标
3. 行动清单
4. 异常与风险
5. 明细附录

决策规则：
- 放量：最佳收入 uplift >= 10%
- 小步放量：最佳收入 uplift >= 5% 且 < 10%
- 暂停/回滚：最佳可用收入 uplift <= -5%
- 补数后判断：核心收入数据缺失或口径不可信
- 继续观察：没有强动作信号

不要做：
- 不要把附录写到前面。
- 不要堆长解释，先回答管理层最关心的问题。
- 不要暴露 cookie、token、webhook、账号信息。

验收标准：
- 2 分钟内能读完。
- 行动清单能直接派给负责人。
- 每个结论都能追溯到指标或数据缺口。

完成后：
- 如果涉及脚本或定时任务，说明检查了哪些文件/命令。
- 如果发现日报格式或自动化规则应更新，指出应写入哪个 skill 或 workflow。
- 如果这次是 Full 模式或暴露了可复用摩擦，按 `daily-work-object-field-test-log.md` 记录一条短 field test。
```
