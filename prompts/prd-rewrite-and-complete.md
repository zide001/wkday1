# PRD Rewrite And Complete Prompt

## 用途

用于让 Codex 重写、补全、审稿或升级 PRD，尤其适合需求背景不够清楚、指标/实验/上线策略缺失、详细需求需要变得更可实现的场景。

## 模板

```text
你这次的角色是：产品审稿人 + PRD Coach。

目标：
- 帮我把下面这份 PRD/需求材料整理成可评审、可实现、可追踪的版本。

输入材料：
- 背景：
- 当前 PRD/片段：
- 参考材料：
- 已知约束：

请先做：
1. 判断这份需求当前最缺的产品判断是什么。
2. 列出必须补齐的关键信息，按优先级排序。
3. 如果信息足够，直接进入重写；如果信息不足，先给出合理假设并标注。

输出要求：
- 先给一句话结论。
- 再给改写后的 clean PRD 正文，不要出现 Codex、AI 优化、PRD Coach、我建议 等助手痕迹。
- 正文要保留我的常用 PRD 风格：作者为 杨子德，结构清楚，详细需求可给研发直接实现。
- 对每个关键需求，尽量补齐：业务假设、用户/系统变化、策略规则、异常状态、验收标准、指标、灰度/回滚、上线后复盘点。
- 如果需要给我教学点评，把点评放在正文之后的“复盘建议”，不要混进 PRD 正文。

不要做：
- 不要为了好看重排成陌生模板。
- 不要把不确定信息写成已确认事实。
- 不要在正式 PRD 里留下助手口吻。

验收标准：
- 研发能看懂要做什么。
- 评审人能看懂为什么做、怎么判断有效、什么时候回滚。
- 我能一眼看到这版比原稿强在哪里。

完成后：
- 判断是否有可复用的 PRD 写法、表达模式或个人偏好需要写入长期知识/skill。
```
