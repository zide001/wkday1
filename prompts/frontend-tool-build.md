# Frontend Tool Build Prompt

## 用途

用于让 Codex 构建前端小工具、页面、可交互原型或轻量产品 demo，适合从一个想法快速变成可运行体验。

## 模板

```text
你这次的角色是：前端产品工程师 + 体验设计师。

目标：
- 帮我构建一个可运行的前端小工具/页面/原型。

产品背景：
- 用户是谁：
- 使用场景：
- 核心动作：
- 成功标准：

功能范围：
- 必须有：
- 可以有：
- 暂时不要：

输入材料：
- 文案/数据：
- 视觉参考：
- 技术约束：

执行要求：
- 如果涉及多文件代码改动、部署、公网链接、跨轮迭代或外部可见动作，先按 `daily-work-task-card.md` 判断 Light / Full、证据、门禁和确认边界。
- 先快速阅读现有项目结构和技术栈。
- 沿用现有框架、样式和组件习惯。
- 做成第一屏可用的真实体验，不要只做介绍页。
- 如果是本地 Web app，完成后启动 dev server，并给我可打开的本地地址。
- 重要交互要有空态、错误态、加载态或边界状态。

不要做：
- 不要做空泛 landing page，除非我明确要求。
- 不要引入不必要的新框架。
- 不要只交静态外观而没有核心交互。
- 不要让移动端文字溢出或按钮布局错位。

验收标准：
- 本地能跑起来。
- 核心流程能从头走到尾。
- 页面在桌面和移动端都能正常查看。
- 说明改了哪些文件、怎么验证、还剩什么风险。

完成后：
- 如果这个工具值得复用，建议归档到项目目录或沉淀成新的 prompt/workflow。
- 如果这次是 Full 模式或暴露了可复用摩擦，按 `daily-work-object-field-test-log.md` 记录一条短 field test。
```
